Git Profolio

A collection of Images and comments from my semester using Git/Github. My Experience with github was fairly limited. I have used the platform to download and experiment with user projects. But i haven't really used it that often. It was only this semester i've really dug into the understanding of how to use github.

Commiting

You commit using the browser or the command line. I preferred the browser because its faster. I commit on the command line if my files are big.

Repo page

This is a hub of where you manage your project(s).You can edit/delete files. Makes the workflow fast and easy.

Github Desktop

Used this from time to time if I wanted to commit something big and didn't want to go through the challenge of the command line.It's a bit confusing to navitgate.

Command line

Command line is notoriously difficult to use. Sometimes its confusing to use and i always have to start from scratch because of a minor mistake.

Creating a repository

Creating a repository wasn't that hard. The downfall of using the browser is the file limit.

Forking a repo

This was a neat feature. It lets other users copy projects without downloading them continuously. I've done this so i can track changes on multiple computers.
